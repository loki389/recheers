# 🔧 解决 Git 仓库连接限制问题

## ⚠️ 错误信息

**"A Git Repository cannot be connected to more than 10 Projects"**

**含义**：一个 Git 仓库最多只能连接到 10 个 Vercel 项目。

---

## 🎯 解决方案

### 方案 1：使用现有项目（推荐）

如果你之前已经创建过项目并连接到这个仓库，直接使用现有项目：

1. **进入 Vercel Dashboard**
   - 访问：https://vercel.com/dashboard

2. **查看项目列表**
   - 在 Dashboard 中查看所有项目
   - 找到你的项目（可能是 `recheers` 或 `wutrecheers` 或类似名称）

3. **使用现有项目**
   - 点击进入现有项目
   - 确认 GitHub 仓库连接正确
   - 直接部署或重新部署

### 方案 2：删除不需要的项目

如果需要创建新项目，先删除不需要的旧项目：

1. **进入 Vercel Dashboard**
   - 访问：https://vercel.com/dashboard

2. **查看所有项目**
   - 找到连接到同一个仓库的项目
   - 识别哪些是不需要的

3. **删除旧项目**
   - 进入不需要的项目
   - 点击 **Settings** → 滚动到底部
   - 找到 **"Delete Project"** 或 **"删除项目"**
   - 确认删除

4. **创建新项目**
   - 删除后，可以创建新项目并连接到仓库

### 方案 3：重命名或使用不同分支

如果项目名称不同，可能只是重命名了项目，实际上还是同一个项目。

---

## 📋 检查步骤

### 1. 检查现有项目

在 Vercel Dashboard 中：
- 查看所有项目列表
- 找到你的项目名称
- 检查 GitHub 仓库连接

### 2. 确认项目设置

进入项目后检查：
- **Settings** → **Git** 部分
- 确认 GitHub 仓库连接正确
- 确认仓库名称匹配

### 3. 直接使用现有项目

如果项目已经存在：
- 不需要创建新项目
- 直接进入现有项目
- 重新部署即可

---

## 💡 建议

**最佳做法**：
- 使用现有项目，而不是创建新项目
- 如果项目名称不合适，可以在项目设置中重命名
- 避免创建多个项目连接到同一个仓库

---

## 🆘 如果找不到现有项目

如果找不到现有项目，可能原因：
1. 项目在不同账户下
2. 项目被删除了
3. 仓库连接在其他 Vercel 账户

**解决**：
- 检查是否登录了正确的 Vercel 账户
- 检查 GitHub 仓库的 Vercel 集成设置

---

**总结**：直接使用现有项目，不要创建新项目。如果确实需要创建新项目，先删除不需要的旧项目。
